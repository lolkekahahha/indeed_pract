# Автоматизированное тестирование сайта secby.ru

## Структура проекта
```
project/
├── tests/ # Директория с тестами
│ ├── test_auth.py # Тесты аутентификации
│ ├── test_profile.py # Тесты профиля пользователя
│ └── test_admin.py # Тесты административных функций
├── api_client.py # API клиент для взаимодействия с сервером
├── config.py # Конфигурация из .env
├── conftest.py # Pytest фикстуры
├── requirements.txt # Зависимости
├── .env.example # Пример конфигурации
└── README.md # Документация
```

## Компоненты проекта

### API Client (`api_client.py`)
Основной класс для взаимодействия с API:

- `APIClient(base_url)` - инициализация клиента
- `login(username, password)` - аутентификация
- `register(username, password, email)` - регистрация
- `get_my_profile()` - получение своего профиля
- `update_my_profile(data)` - обновление профиля
- `get_all_profiles()` - получение всех профилей (админ/модератор)
- `update_role(account_id, role_name)` - обновление роли пользователя

### Конфигурация (`config.py`)
Загружает переменные окружения из `.env` файла:
- `BASE_URL` - базовый URL API
- `ADMIN` - учетные данные администратора
- `MODERATOR` - учетные данные модератора
- `TEST_USER` - учетные данные тестового пользователя

### Фикстуры Pytest (`conftest.py`)

| Фикстура | Область видимости | Описание |
|----------|------------------|----------|
| `api_client` | session | Базовый клиент без авторизации |
| `authenticated_client` | function | Клиент с авторизованным тестовым пользователем |
| `admin_client` | function | Клиент с правами администратора |
| `moderator_client` | function | Клиент с правами модератора |

## Тесты

### `test_auth.py` - Тесты аутентификации
- Успешный вход администратора и модератора
- Обработка неверного пароля
- Вход несуществующего пользователя
- Доступ к защищенным эндпоинтам без токена
- Формат Authorization header

### `test_profile.py` - Тесты профиля пользователя
- Получение своего профиля
- Обновление профиля (с fallback на другой эндпоинт)
- Доступ без авторизации
- Валидация пустых полей

### `test_admin.py` - Административные тесты
- Получение всех профилей (админ и модератор)
- Обновление ролей пользователей
- Проверка запрета на изменение ролей для модератора
- Получение заметок (доступ по ролям)

## Настройка окружения

1. Создайте виртуальное окружение и активируйте его:
```
python -m venv .venv
.venv\Scripts\activate
```

2. Установите зависимости:
```
pip install -r requirements.txt
```

3. Скопируйте шаблон конфигурации и заполните данные:
```
cp .env.example .env
```

## Запуск тестов

```
# Запуск всех тестов
pytest tests/ -v

# Запуск конкретного модуля
pytest tests/test_auth.py -v

# Запуск с подробным выводом (включая print)
pytest tests/ -v -s
```
